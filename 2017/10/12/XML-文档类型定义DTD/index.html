XML-文档类型定义-DTD

Veröffentlicht am 2017-10-12

DTD 简介

文档类型定义（DTD）定义了 XML 文档中可用的合法属性，可以使用 DTD 指定可以在文档中存在的元素、元素可以具有的属性、在元素内部子元素的层次结构以及元素在整个文档中出现的顺序。

符号 含义
? 元素不出现或出现一次
* 元素不出现或出现多次
+ 元素出现一次以上
无符号 元素只出现一次

属性

格式：<!ATTLIST element-name attribute-name attribute-type attribute-value>
即是 <!ATTLIST 元素名 属性名 属性类型 属性值>
属性类型如下：

类型 含义
CDATA 纯文本
Enumerated 取值是列表中的一个值
ID 值为唯一的 id
IDREF 值为另外一个元素的 id
IDREFS 值为其他 id 的列表
NMTOKEN 值为合法的 XML 名称
NMTOKENS 值为合法的 XML 名称的列表
ENTITY 值是一个实体
ENTITIES 值是一个实体列表
NOTATION 此值是符号的名称

属性特点如下：

特点 含义
#REQUIRED 必须赋值

此时考号属性就只能取固定值10001

实体

内部调用格式：<!ENTITY entity-name “entity-value”>
外部调用格式：<!ENTITY entity-name SYSTEM “URI/URL”>
可以理解为宏定义，即是多次在 XML 文档中使用同一个字符
预定义实体
XML 文档中预先保留的实体，包括尖括号（左和右）、引用符号&、单引号、双引号

注意：调用时一定要包括三部分，&、字符、分号

自定义实体

此时在 XML 文档中就可以使用 &name; 来表示”hello world”了

总结

通过 DTD，每一个 XML 文件均可携带一个有关其自身格式的描述
通过 DTD，独立的团体可一致地使用某个标准的 DTD 来交换数据
而应用程序也可使用某个标准的 DTD 来验证从外部接收到的数据
还可以使用 DTD 来验证自身的数据
但 XML Schema 很快会将 DTD 取而代之，用在大部分的网络应用程序中。
